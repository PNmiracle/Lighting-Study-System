# PRD：留学机构学生与选导管理系统

## 1. 项目信息

| 项目 | 内容 |
|------|------|
| **项目名称** | study_abroad_mgmt |
| **语言** | 中文 |
| **技术栈** | Vite + React + MUI + Tailwind CSS（前端）；PostgreSQL（独立数据库） |
| **原始需求** | 构建留学机构学生和机构老师的管理系统网站，核心业务为"选导"流程——选导老师帮学生搜索、挑选海外 PhD 导师，整理成导师列表推给学生，学生标记意向后反馈给老师。脱离 Vika 多维表格，使用独立 PostgreSQL 数据库。 |

---

## 2. 产品目标

1. **替代 Vika 多维表格**，将分散在多张 Vika 表中的选导流程统一到一个系统中，实现数据集中管理、角色权限控制和流程标准化。
2. **提升选导效率**，为选导老师提供紧凑表格工作台批量管理导师，为学生提供卡片式浏览体验标记意向，形成"搜索→推荐→反馈→调整"的高效闭环。
3. **沉淀导师数据库**，将导师信息从"每学生一张表"的重复录入变为可复用的导师库，减少重复劳动并支持跨学生复用。
4. **全流程进度可视**，机构管理员可查看老师工作进度、学生选导进展，实现机构层面的统筹管理。

---

## 3. 用户角色定义

| 角色 | 核心职责 | 权限范围 |
|------|---------|---------|
| **机构管理员（Admin）** | 管理选导老师和学生账号；分配学生给选导老师；查看全局进度和统计数据 | 全系统读写权限；管理所有老师和学生 |
| **选导老师（Teacher）** | 搜索海外 PhD 导师；为分配给自己的学生创建导师推荐列表；填写导师备注与匹配度评价；接收并处理学生反馈 | 管理自己名下学生的导师列表；读写导师数据库 |
| **学生（Student）** | 浏览选导老师推荐的导师卡片；标记选导意向（想联系/备选/跳过）；查看导师详情链接 | 只能查看自己被推荐的导师列表；标记意向后该导师记录锁定 |

## 5. 需求池

### P0 — 必须实现（MVP 核心）

| 编号 | 模块 | 需求描述 | 验收标准 |
|------|------|---------|---------|
| P0-01 | 用户认证 | 支持管理员/老师/学生三种角色登录 | 不同角色登录后跳转各自工作台；未登录不可访问任何页面 |
| P0-02 | 用户管理 | 管理员可创建、编辑、禁用老师和学生账号 | 账号创建后可立即登录；禁用后无法登录 |
| P0-03 | 师生分配 | 管理员可将学生分配给选导老师 | 一个学生同时只分配给一位老师；老师可在工作台看到分配的学生 |
| P0-04 | 导师录入 | 老师可手动录入导师信息（姓名/学校/排名/院系/主页/邮箱/申请链接等） | 所有字段可保存到导师数据库；必填字段校验 |
| P0-05 | 导师推荐 | 老师可为学生创建导师推荐列表，填写备注和匹配度标识 | 每条推荐关联一个学生和一个导师；备注包含职称+研究方向+匹配度标识 |
| P0-06 | 学生浏览 | 学生可查看被推荐的导师列表（卡片视图） | 仅显示分配给自己的导师；卡片展示关键信息 |
| P0-07 | 选导意向 | 学生可通过三态按钮标记意向（想联系/备选/跳过） | 标记后状态实时更新；标记"想联系"或"备选"后该记录锁定不可修改 |
| P0-08 | 意向反馈 | 老师可在工作台表格中看到学生的意向反馈 | 学生标记后老师端实时可见；表格中显示意向状态列 |
| P0-09 | 导师数据库 | 系统维护独立的导师信息库，支持跨学生复用 | 同一导师可被推荐给多个学生；搜索时可匹配已有导师 |

| | notes | TEXT | 备注全文 | 备注 |
| | match_level | ENUM | high(建议多看看) / medium(可以备选) | （从备注解析） |
| | student_intent | ENUM | want_contact / backup / skip / null | 选导意向 |
| | intent_locked | BOOLEAN | 意向锁定状态 | （Vika 行锁定） |
| | created_at | TIMESTAMP | 创建时间 | — |
| | updated_at | TIMESTAMP | 更新时间 | — |

### 7.3 关键设计决策

1. **导师库与推荐记录分离**：Supervisor 表是全局可复用的导师数据库，SupervisorAssignment 是"某学生×某导师"的推荐记录（含备注和意向）。同一导师可被推荐给多个学生，避免重复录入。
2. **备注结构化**：Vika 中的备注是自然语言文本，系统将其拆解为 `title`（职称）、`research_area`（研究方向）、`match_level`（匹配度枚举）三个结构化字段，同时保留 `notes` 原文。
3. **意向锁定机制**：学生标记"想联系"或"备选"后，`intent_locked = true`，该推荐记录不可被学生修改（对应 Vika 行锁定）。老师仍可编辑。

## 8. 待确认问题

| 编号 | 问题 | 影响范围 | 建议 |
|------|------|---------|------|
| Q-01 | 学生标记意向后是否完全不可撤销？还是允许学生在一定条件下修改？ | 选导意向功能 | 建议允许老师端解锁后学生重新标记 |
